# Changelog Entry Template

When making changes, update CHANGELOG.md under the [Unreleased] section:

```markdown
## [Unreleased]

### Added
- Description of new feature

### Changed
- Description of changed feature

### Fixed
- Description of bug fix

### Security
- Description of security fix
```

## Commit Message Format

Use conventional commits to make changelog updates easier:

- `feat:` - New feature (adds to "Added")
- `fix:` - Bug fix (adds to "Fixed")
- `chore:` - Maintenance (usually no changelog entry)
- `docs:` - Documentation (usually no changelog entry)
- `refactor:` - Code refactoring (adds to "Changed")
- `perf:` - Performance improvement (adds to "Changed")
- `security:` - Security fix (adds to "Security")

Example:
```bash
git commit -m "feat: add product search functionality"
# Then update CHANGELOG.md under "Added"
```
